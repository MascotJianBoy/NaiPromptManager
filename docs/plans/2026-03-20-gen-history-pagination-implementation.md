# 历史页面分页改造实现计划

## 目标
实现智能分页 + 缓存混合方案，解决当前翻页逻辑的四个痛点：
1. ✅ 支持前后翻页和快速跳转
2. ✅ 显示准确的页数信息
3. ✅ 保留已加载的历史数据
4. ✅ 添加页码按钮

## 技术方案

### 1. 状态管理增强
- 新增 `currentPage` 状态跟踪当前页
- 新增 `totalPages` 状态计算总页数
- 新增 `pageCache` 对象缓存各页数据
- 保留 `items` 状态用于渲染

### 2. 缓存策略
- **缓存范围**：当前页 ± 1 页（共3页）
- **缓存结构**：`pageCache: { [page: number]: LocalGenItem[] }`
- **缓存清理**：超出范围的页面从缓存中移除
- **内存控制**：最多缓存 60 条记录（3页 × 20条/页）

### 3. 核心方法
- `goToPage(page: number)` - 跳转到指定页
- `loadPage(page: number)` - 加载指定页数据
- `updateCache(page: number)` - 更新缓存策略
- `getPageData(page: number)` - 获取页面数据（优先从缓存）

### 4. UI 组件
- 页码导航栏：首页、上一页、页码按钮、下一页、末页
- 页码输入框：支持直接输入页码跳转
- 页数信息：显示"第 X 页 / 共 Y 页"
- 刷新按钮：重新加载当前页
